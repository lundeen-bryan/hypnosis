Class of Problems → Class of Solutions (Bill O’Hanlon)

A top‑to‑bottom visual of Handout 5.1: derive the problem class, evoke the matching solution class, create a specific intervention, then transfer it back to the client’s real context.

1Specific Presenting Problem

Name the behavior in concrete, observable terms—what you could see on video.

  • Ask: What exactly happens? how often? when?
  • Write one line: “After work I eat sweets until uncomfortably full.”

2Derive the Class of Problem

Ask: What keeps this going right now? Choose the dominant class.

  • Classes: Excess • Absence/Deficit • Instability • Attempted‑Solution • Frame Mismatch • Secondary Gain
  • Example: Overeating for relief ⇒ Excess (stress/arousal)

3Match & Evoke the Class of Solution

Pick the matching move and plan to evoke an experience of it (not a lecture).

  • Map: Excess → Reduce / Redirect • Absence → Elicit/Increase • Instability → Stabilize/Regulate • Attempted‑Solution → Do Something Different • Frame Mismatch → Change Frame • Secondary Gain → Protect Function
  • Example: Evoke experiences of stress decreasing without food.

4Create a Specific Intervention

Deliver the experience via any of the following (Erickson’s “interpersonal evocation” lives here):

  • Analogy / story • Anecdote • Trance phenomenon • Behavioral task • Interpersonal move
  • Example: Revivify a safe “waiting” experience; evoke spontaneous breath‑ease; brief task: “notice first sign, let the body soften for 10s.”

5Transfer to Real‑Life Context

Bridge it back: link the new experience to the actual situation where it helps.

  • Prompt: “Sometimes this same easier breathing can show up at the kitchen counter before the first bite.”
  • Note: If nothing changes in context, re‑check the problem class.
